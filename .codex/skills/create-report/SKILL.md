# create-report

Markdownタスク情報からSlack投稿本文を作成する。

## モード

- `current_status`: 現在時点の状況報告。
- `daily_end`: 終業時の完了・持ち越し報告。

モード未指定時は `current_status` を使う。

## 参照優先順位

1. 最新のTODO/状況Markdown
2. 現ターンでのユーザー明示指示
3. 利用可能なGitHubアサイン情報

## 出力構成

指定がなければ次の構成を使う。

```md
【現在の状況】
- ...

【今持っているタスク】
- ...

【今日やるタスク】
- ...

【今後やるタスク】
- ...

【懸念・ブロッカー】
- ...
```

## 手順

1. 最新のタスクMarkdownを読み取る。
2. 内部メモを除いた、短く具体的な箇条書きを作る。
3. 各行は行動と状態が分かる表現にする。
4. プレビュー本文を返し、送信は明示指示待ちにする。

## 注意点

- このskillから自動送信しない。
- 報告は読みやすい長さに抑える（通常は合計5〜15行程度）。
- 不確実な情報は推測せず `?` を付ける。
